Shortest path algorithm

1. Dijkstra

Given a graph G=(V, E) with positive edge and a source vertex s, how to find the shortest path/distance from s to any vertex?

Dijkstra’s algorithm works as follow. Given a Known set K and distrance function Ds(x)D_s(x)Ds​(x) for x∈Kx \in Kx∈K, initialized with K={s}K=\{s\}K={s}, Ds(s)=0D_s(s) = 0Ds​(s)=0.

By induction, suppose we have already known shortest distance between sss and vertices in KKK. Suppose vertices in set NNN are vertices adjacent to vertices in set KKK. We can define the distance dS(y)d_S(y)dS​(y) for y∈Ny\in Ny∈N by simply stretching from KKK to NNN, namely dS(y)=min⁡x∈KDS(x)+dxyd_S(y) = \min_{x \in K} D_S(x) + d_{xy}dS​(y)=minx∈K​DS​(x)+dxy​.

If y∗=argminy∈NdS(y)y^*= argmin_{y \in N} d_S(y)y∗=argminy∈N​dS​(y), the we claim Ds(y∗)=ds(y∗)D_s(y^*)=d_s(y^*)Ds​(y∗)=ds​(y∗). The reason is fairly simple: if there is a better path s – x -|- y – y*, where {s,x}⊆K\{s, x\} \subseteq K{s,x}⊆K, {y,y∗}⊆N\{y, y^*\} \subseteq N{y,y∗}⊆N, obviously

Ds(y∗)=Ds(x)+d(x,y)+Dy(y∗)=Ds(y)+Dy(y∗)≥Ds(y∗)(1)\begin{aligned} D_s(y^*) &= D_s(x) + d(x,y) + D_y(y^*) \\ &= D_s(y) + D_y(y^*) \\ &\ge D_s(y^*) \end{aligned} \tag{1} Ds​(y∗)​=Ds​(x)+d(x,y)+Dy​(y∗)=Ds​(y)+Dy​(y∗)≥Ds​(y∗)​(1)

By premisis of the problem, the equation holds true only when Ds(y)=Ds(y∗)D_s(y)=D_s(y^*)Ds​(y)=Ds​(y∗) and Dy(y∗)=0D_y(y^*)=0Dy​(y∗)=0. Obviously, this can take place only when y=y∗y=y^*y=y∗. Therefore we update by letting K′=K+{y∗}K'=K+\{y^*\}K′=K+{y∗}, Ds(y∗)=dS(y∗)D_s(y^*)=d_S(y^*)Ds​(y∗)=dS​(y∗)

// Dijkstra with heap
#include<cstdio>

    }
};
// Adjacent graph

int dis[maxn];  // D_S(x)
bool vis[maxn]; // vis[x] = 1 if x in K

int main(){
    int n, m;
    while( scanf("%d %d",&n, &m) && n + m){
        for(int i = 0; i <=n; ++i){

            vis[i] = 0;
            vec[i].clear();
        }
        int s, t, d;
        for(int i = 0; i < m; ++i){
            scanf("%d %d %d %d", &s, &t, &d, &p);
            vec[s].push_back( (edge){t, d} );
            vec[t].push_back( (edge){s, d} );
        }
        int st, ed;
        cin >> st >> ed;
 
        // start
        priority_queue<edge> que;
        que.push( (edge){st, 0} );

        while(!que.empty()){
            edge top = que.top();
            que.pop();
            // If y is not in K, it's y*
            if( !vis[top.x] ){
                vis[top.x] = 1;         // Add to K
                dis[top.x] = top.d;     // D_S(x) = d_S(x)

                // Update N and d_S(y)
                for(int i = 0; i < vec[top.x].size(); ++i){
                    edge &e = vec[top.x][i];
                    if( vis[edge.x])
                        continue;
                    if( top.d + e.d < dis[e.x] )
                        que.push( (edge){e.x, top.d + e.d} );
                }
            }
            if(vis[ed])
                break;
        }
        printf("%d %d\n", dis[ed]);
    }
}

How to find shortest path ?

Introduce an extra array pa[MAXN], when y∗y^*y∗ is added into KKK, let pa[y∗]pa[y^*]pa[y∗] be the closet x∈Kx\in Kx∈K.
Then we can backtrack the path from ed back to the beginning.

2. Bellman-ford

If the graph contains negative edges, the equation (1) won’t hold true any longer, since dS(y,y∗)d_S(y, y^*)dS​(y,y∗) is no longer non-negative.

Bellman-ford is proposed to solve this problem. It’s more general than Dijkstra. The pseudo-code is:

dis[|V|] = {INF, INF, ..., INF}
dis[S] = 0
for i = 1 to |V|-1
    for e in E
        dis[e.ed] = min( dis[e.ed], dis[e.s] + e.dis)

Proof: ∀vi∈V\forall v_i \in V∀vi​∈V, if there are shortest paths (implies viv_ivi​ is reachable from SSS), consider the shortest path Pi=[ϕ1,ϕ2,...,ϕk]P_i=[\phi_1, \phi_2, ..., \phi_k]Pi​=[ϕ1​,ϕ2​,...,ϕk​] from SSS to viv_ivi​, which means ϕ1=S\phi_1=Sϕ1​=S and ϕk=vi\phi_k=v_iϕk​=vi​.

No vertix vj∈Vv_j \in Vvj​∈V can appear twice in the path. Otherwise, there must be a path P′=[Φ1,...,Φk]P'=[\Phi_1, ..., \Phi_k]P′=[Φ1​,...,Φk​].

  • If the length of P′P'P′ is positive, then obviously we chan trim the path to shorten length, leading to contradiction that PiP_iPi​ is not the shortest path from SSS to viv_ivi​.
  • If the length of P′P'P′ is 0, it won’t hurt the fact that this is the shortest path. We can cherry-pick the simplified path without repeated vertices, the two shortest paths have equal shortest distances.
  • If the length of P′P'P′ is negative, a negative loop is promised to exist. In that case, no shortest paths can exist.

Now we can safely claim that any shortest path from SSS contains # of vertices less than ∣V∣|V|∣V∣. In fact, there are at most |V|-1 edges between SSS and termination. In the first round of brute relaxation for all edges, for Φ1\Phi_1Φ1​, DS(Φ1)=dS(Φ1)D_S(\Phi_1) = d_S(\Phi_1)DS​(Φ1​)=dS​(Φ1​) is attained when edge e=(S,Φ1)e=(S,\Phi_1)e=(S,Φ1​) is relaxed (Otherwise there must be a shorter path from SSS to Φ1\Phi_1Φ1​). By induction we know that DS(Φ2)=DS(Φ1)+d(Φ1,Φ2)=dS(Φ2)D_S(\Phi_2)=D_S(\Phi_1) + d(\Phi_1, \Phi_2)=d_S(\Phi_2)DS​(Φ2​)=DS​(Φ1​)+d(Φ1​,Φ2​)=dS​(Φ2​), then so on forward.

If relaxation is made for all edges for |V|-1 rounds, all shortest paths will be completely updated, then dS(v)d_S(v)dS​(v) for all v∈Vv\in Vv∈V turns out to be equal to DS(v)D_S(v)DS​(v). If dS(v)=∞d_S(v)=\inftydS​(v)=∞, obvious it means vvv is unreachable.

The complexity is O(∣V∣⋅∣E∣)O(|V|\cdot|E|)O(∣V∣⋅∣E∣).

3. Floyd algorithm

Floyd algorithm is the generalization of Bellman-ford algorithm for all pais of points. We can prove the theorem by recursion. Consider the (n2){ n \choose 2}(2n​) pairs. For i→ji\rightarrow ji→j, if the shortest path exists, let the path be Pij={v1,v2,...,vm}P_{ij}=\{v_1, v_2, ..., v_m\}Pij​={v1​,v2​,...,vm​}, where v1=iv_1=iv1​=i and vm=jv_m=jvm​=j. Define max(Pij)=max⁡l=1mvlmax(P_{ij})=\max_{l=1}^m v_lmax(Pij​)=maxl=1m​vl​.

Now we prove the correctness by recursion. Given k, suppose we have already found shortest distance for all shortest paths that includes vertex with index less than k−1k-1k−1. Next, consider all pairs of (i,j)(i,j)(i,j) with max(Pij)=kmax(P_{ij}) = kmax(Pij​)=k. Let’s denote k+k^+k+ for the vertex in the right-hand side of kkk in path PijP_{ij}Pij​, and k−k^-k− for the reverse. There are two cases:

  • k is on double-sides. Then D(i,k)=D(i,k−)+D(k−,k)D(i,k)=D(i, k^-) + D(k^-, k)D(i,k)=D(i,k−)+D(k−,k) or D(k,i)=D(k,k+)+D(k+,j)D(k,i)=D(k, k^+) + D(k^+, j)D(k,i)=D(k,k+)+D(k+,j). D(k−,k)D(k^-, k)D(k−,k) or D(k,k+)D(k, k^+)D(k,k+) are edges, known in advance. The rest parts are obviously subproblems with max(P′)<kmax(P')<kmax(P′)<k. By recursion, we can find the shortest distance.
  • k is in the middle. Then D(i,j)=D(i,k−)+D(k−,k)+D(k,k+)+D(k+,j)D(i,j) = D(i,k^-) + D(k^-, k) + D(k, k^+) + D(k^+, j)D(i,j)=D(i,k−)+D(k−,k)+D(k,k+)+D(k+,j). D(k−,k)D(k^-, k)D(k−,k) and D(k,k+)D(k, k^+)D(k,k+) are edges known in advance, and the rest parts are subproblem with max(P′)<kmax(P')<kmax(P′)<k. Obviously, we can find the shortest distance.

Finally, consider the terminal condition. k=1k=1k=1 corresponds to the only path P11P_{11}P11​. It’s surely solved, as D(1,1)=0D(1,1)=0D(1,1)=0 (otherwise there would be negative circles.)

In practice, we can write the code by induction, from k=1k=1k=1 to ∣V∣|V|∣V∣.

dp[|V|][|V|] = G[|V|][|V|]       
for k = 1 to |V|              // Bottom up for shortest paths max(P) <= k
    for i = 1 to |V|            
        for j = 1 to |V|        
            if dis[k][j] != INF
                dp[i][j] = min(dp[i][j], dp[i][k] + dp[k][j])

Key to understanding: Solvable paths are readily solved, don’t be confused by asking those pairs not currently solvable.
